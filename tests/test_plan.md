# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] El programa corre en su caso nominal.
- [ ] Se probó al menos una entrada inválida.
- [ ] Se verificó al menos un caso límite simple.
- [ ] Se registraron resultados en `docs/plan_de_pruebas.md`.

## 2) Checklist de documentación
- [ ] `docs/propuesta.md` está completo.
- [ ] `docs/caso_de_uso.md` está completo.
- [ ] `docs/estructura_repositorio.md` refleja la organización real.
- [ ] `docs/plan_de_pruebas.md` contiene casos y resultados.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` funciona sin usar red.
- [ ] Existe `src/main.<ext>` acorde al lenguaje elegido.
- [ ] El flujo principal puede explicarse paso a paso.

## 4) Checklist de entrega final
- [ ] El alcance se mantuvo pequeño y viable.
- [ ] No se usaron frameworks ni dependencias complejas.
- [ ] No se usaron bases de datos, nube, Docker ni APIs externas.
- [ ] El repositorio está ordenado y listo para GitHub Classroom.
