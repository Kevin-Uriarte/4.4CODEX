# Checklist Breve de Pruebas y Entrega

## 1) Checklist de pruebas mínimas
- [ ] El programa corre al menos en un caso válido.
- [ ] Se probó al menos un caso con error de entrada.
- [ ] Se documentó el resultado esperado y el obtenido.

## 2) Checklist de documentación
- [ ] `docs/propuesta.md` completo.
- [ ] `docs/caso_de_uso.md` completo.
- [ ] `docs/plan_de_pruebas.md` con casos definidos.
- [ ] Alcance mínimo y límites claramente definidos.

## 3) Checklist de ejecución
- [ ] `scripts/run.sh` tiene permisos de ejecución (si aplica).
- [ ] Existe `src/main.<ext>` acorde al lenguaje elegido.
- [ ] Se probó la ejecución desde la raíz del repositorio.

## 4) Checklist de entrega final
- [ ] Repositorio ordenado y sin archivos basura.
- [ ] Commits con mensajes claros.
- [ ] README revisado y consistente con el contenido.
- [ ] Actividad lista para revisión en GitHub Classroom.
