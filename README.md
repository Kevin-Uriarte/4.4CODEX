# Actividad GitHub Classroom: Propuesta de Práctica Temática (Proyecto Pequeño)

## Descripción general
En esta actividad vas a diseñar una **propuesta de práctica temática pequeña** y viable para desarrollarla en terminal, usando uno de los lenguajes permitidos. La prioridad es que construyas una idea clara, bien documentada y con alcance realista para un proyecto corto.

## Objetivo de aprendizaje
Al finalizar esta actividad, serás capaz de:
- Definir un problema concreto y proponer una solución pequeña.
- Justificar técnicamente la elección de lenguaje.
- Organizar un repositorio con estructura limpia y mantenible.
- Documentar caso de uso, alcance y plan de pruebas antes de programar.
- Implementar (opcionalmente) un prototipo mínimo funcional.

## Lenguajes permitidos
- ARM64 Assembly
- C
- Python
- Bash

## Reglas para mantener el proyecto pequeño
- Define un solo caso de uso principal.
- Limita las funciones del prototipo a lo esencial (MVP).
- Evita interfaces gráficas, frameworks y arquitecturas complejas.
- No uses bases de datos, nube, contenedores ni APIs externas.
- Máximo recomendado: 1 archivo principal de código y scripts auxiliares mínimos.
- Si eliges ARM64 Assembly, enfócate en un programa muy pequeño (entrada/salida básica y operaciones simples).

## Entregables esperados
1. `docs/propuesta.md` completo.
2. `docs/caso_de_uso.md` completo.
3. `docs/estructura_repositorio.md` revisado y adaptado si aplica.
4. `docs/plan_de_pruebas.md` con casos definidos.
5. `tests/test_plan.md` con checklist final.
6. Prototipo mínimo en `src/main.<ext>` (opcional pero recomendado).

## Instrucciones para el estudiante
1. Lee todo el repositorio antes de modificar archivos.
2. Completa primero la documentación en `docs/`.
3. Delimita el alcance mínimo y lo que queda fuera.
4. Diseña pruebas manuales simples en `docs/plan_de_pruebas.md`.
5. Si el tiempo lo permite, implementa un prototipo pequeño en `src/`.
6. Ejecuta tu flujo con `scripts/run.sh` y registra resultados.
7. Entrega el repositorio ordenado y con evidencias claras de planeación.

## Ejemplos de posibles temas
- **Mini Toolkit en ARM64**
- **Asistente de Estudio en Terminal**
- **Reporteador de Información del Sistema**
- **Organizador de Archivos**
- **Juego de Aprendizaje en Línea de Comandos**

## Criterios generales de evaluación
- Claridad y calidad de la propuesta.
- Viabilidad técnica del alcance.
- Justificación del lenguaje elegido.
- Calidad del caso de uso y de los flujos.
- Coherencia del plan de pruebas.
- Organización del repositorio.
- (Opcional) Correcto funcionamiento del prototipo mínimo.

## Nota importante
**Primero se documenta y justifica la idea; después, de forma opcional, se implementa un prototipo pequeño.**
